# 前端页面线框说明 v1.0

## 1. 页面目标

在单页面中完成以下能力：
- 输入 URL 创建爬虫任务。
- 使用命令行控制任务。
- 实时查看待爬取/已爬取/失败队列。
- 查看清洗前后结果。
- 查看实时事件日志与错误信息。


移动端交互原则：
- 以 Tab 切换复杂区域，避免并排多列。
- 命令输入框固定底部或顶部，保证可快速操作。

## 4. 页面组件与职责

### 4.1 顶部栏 `HeaderBar`
- 显示系统名、API 连接状态、当前鉴权信息。
- 状态灯：`green=正常`，`yellow=降级`，`red=异常`。

### 4.2 任务创建区 `TaskSubmitBar`
- 输入：`url`、`limit`、`depth`。
- 按钮：`开始爬取`、`停止当前任务`。
- 校验失败时显示明确错误（如 URL 非法、参数越界）。

### 4.3 命令控制台 `CommandConsole`
- 输入命令并回车执行。
- 支持历史命令上下键浏览。
- 支持快捷命令提示（输入 `help`）。
- 展示每条命令的回显、错误码、request_id。

### 4.4 任务总览 `TaskOverview`
- 展示任务聚合统计：运行中、排队中、失败、已完成。
- 展示当前选中任务 `task_id`、状态、进度条、耗时。

### 4.5 队列面板 `QueueBoard`
- 三列/三 Tab：`待爬取`、`已爬取`、`失败`。
- 每项显示：URL、状态、重试次数、最后更新时间。
- 支持筛选与分页。

### 4.6 结果面板 `ResultTable`
- 视图切换：`原始数据` / `清洗数据`。
- 支持搜索、排序、分页。
- 支持导出 JSON/CSV。

### 4.7 事件流 `EventStream`
- 实时滚动展示 `task_created` 等事件。
- 可按等级过滤：`INFO/WARN/ERROR`。
- 可按 `task_id` 过滤。

## 5. 关键交互流程

### 5.1 创建任务流程
1. 用户输入 URL 与参数，点击“开始爬取”。
2. 前端调用 `POST /v1/crawl/submit`。
3. 成功后保存 `task_id`，并自动订阅 `/v1/events/stream`。
4. 页面更新任务总览与队列面板。

### 5.2 命令执行流程
1. 用户输入命令并提交。
2. 前端调用 `POST /v1/command`。
3. 回显输出到命令区，并根据返回刷新相关模块。
4. 若命令失败，显示错误码和建议命令。

### 5.3 实时更新流程
1. 事件流收到消息后按 `event_type` 路由更新组件。
2. 若连接断开，进入轮询兜底（每 3 秒）：
- `GET /v1/tasks/{task_id}`
- `GET /v1/tasks/{task_id}/queue`
- `GET /v1/tasks/{task_id}/results`
3. 连接恢复后回到事件驱动模式。

## 6. 页面状态与提示规范

任务状态展示：
- `pending`：灰色
- `running`：蓝色
- `paused`：橙色
- `success`：绿色
- `failed`：红色
- `stopped`：深灰

空状态：
- 队列为空：显示“暂无数据，请先创建任务”。
- 结果为空：显示“尚无结果，等待抓取或清洗完成”。

错误提示：
- 展示错误码 + 简短描述 + 处理建议。
- 示例：`1002 URL 不允许，请检查协议或内网地址限制。`

## 7. 前端数据结构建议（TypeScript）

```ts
type TaskStatus = "pending" | "running" | "paused" | "stopped" | "success" | "failed";
type QueueState = "pending" | "running" | "done" | "failed" | "canceled";

interface ApiResp<T> {
  code: number;
  message: string;
  request_id: string;
  data: T;
  meta?: Record<string, unknown>;
}
```

```ts
interface TaskSummary {
  task_id: string;
  status: TaskStatus;
  progress: number;
  total_count: number;
  done_count: number;
  failed_count: number;
  clean_done_count: number;
}
```

```ts
interface QueueItem {
  id: number;
  url: string;
  state: QueueState;
  retry_count: number;
  next_run_at?: string;
  last_error?: string;
  updated_at: string;
}
```

- 可在命令区执行控制命令并收到回显。
